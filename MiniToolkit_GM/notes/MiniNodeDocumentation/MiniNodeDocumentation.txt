# MiniNode - Function Documentation

## Overview

THIS DOCUMENTATION WAS WRITTEN BY AI, IT MAY CONTAIN ERRORS

**MiniNode** is a modular, component-based user interface (UI) system for GameMaker. It facilitates UI hierarchy creation, transform inheritance (position, scale, rotation, alpha), deferred drawing, event-driven architecture, and automatic keyboard/gamepad/mouse directional navigation.

**Author:** M. Neet - https://github.com/mneet/MiniToolkit-GM
**Version:** 1.0.0

## Enums

### NODE_ORIGIN
Defines the anchor point of a node for positioning, scaling, and rotation.
- `NODE_ORIGIN.TOP_LEFT`
- `NODE_ORIGIN.TOP_CENTER`
- `NODE_ORIGIN.TOP_RIGHT`
- `NODE_ORIGIN.MIDDLE_LEFT`
- `NODE_ORIGIN.MIDDLE_CENTER`
- `NODE_ORIGIN.MIDDLE_RIGHT`
- `NODE_ORIGIN.BOTTOM_LEFT`
- `NODE_ORIGIN.BOTTOM_CENTER`
- `NODE_ORIGIN.BOTTOM_RIGHT`

### NODE_HFLOW
Horizontal flow direction for `MiniNodeContainer` auto-layout.
- `NODE_HFLOW.LEFT` (Right to Left)
- `NODE_HFLOW.RIGHT` (Left to Right)

### NODE_VFLOW
Vertical flow direction for `MiniNodeContainer` auto-layout.
- `NODE_VFLOW.UP` (Bottom to Top)
- `NODE_VFLOW.DOWN` (Top to Bottom)

### NODE_AXIS
Primary fill direction for `MiniNodeContainer` auto-layout.
- `NODE_AXIS.HORIZONTAL` (Fills row first, then adds columns/rows)
- `NODE_AXIS.VERTICAL` (Fills column first, then adds columns/rows)

### MND_NAV_INPUT
Global navigation mode.
- `MND_NAV_INPUT.BUTTON` (Keyboard/Gamepad input)
- `MND_NAV_INPUT.CURSOR` (Mouse/Touch hover input)

## Constructors

### MiniNode(_id, _origin)
Creates the base element with transform, rendering, and processing components.
- `_id` — Unique string identifier
- `_origin` — Optional `NODE_ORIGIN` (default: `MIDDLE_CENTER`)
- **Returns:** Struct (MiniNode instance)

### MiniNodeCanvas(_name, _origin)
Organizational root node that can be registered in the NodeManager.
- `_name` — Unique string identifier
- `_origin` — Optional `NODE_ORIGIN` (default: `TOP_LEFT`)
- **Returns:** Struct (MiniNodeCanvas instance)

### MiniNodeContainer(_name, _columns, _origin)
A layout node that automatically arranges children in a list or grid.
- `_name` — Unique string identifier
- `_columns` — Optional columns number (default: `0` / unlimited)
- `_origin` — Optional `NODE_ORIGIN` (default: `MIDDLE_CENTER`)
- **Returns:** Struct (MiniNodeContainer instance)

### MiniNodeSprite(_name, _sprite, _color, _animate, _origin)
A node that renders a sprite with optional frame animation.
- `_name` — Unique string identifier
- `_sprite` — Sprite asset index
- `_color` — Optional blend color (default: `c_white`)
- `_animate` — Optional boolean to auto-animate subimages (default: `false`)
- `_origin` — Optional `NODE_ORIGIN` (default: `MIDDLE_CENTER`)
- **Returns:** Struct (MiniNodeSprite instance)

### MiniNodeText(_name, _text, _font, _color, _origin)
A node that renders text with custom alignment and optional wrapping.
- `_name` — Unique string identifier
- `_text` — String text content
- `_font` — Font asset index
- `_color` — Color constant
- `_origin` — Optional `NODE_ORIGIN` (default: `MIDDLE_CENTER`)
- **Returns:** Struct (MiniNodeText instance)

### MiniNodeButton(_name, _origin)
Interactive button node that handles selection, triggers action events, and automatically binds a navigator component.
- `_name` — Unique string identifier
- `_origin` — Optional `NODE_ORIGIN` (default: `MIDDLE_CENTER`)
- **Returns:** Struct (MiniNodeButton instance)

## Core Components
Every `MiniNode` contains these helper components initialized inside constructor:
- `renderer` — `MiniNodeRenderer` struct handling drawing routines.
- `processor` — `MiniNodeProcessor` struct handling lifecycle events and custom processes.
- `navigator` — `MiniNodeNavigator` struct (default `noone` unless interactive button).
- `nav_module` — `MiniNodeNavCenter` struct (default `noone` unless navigator hub).
- `transform` — `NodeTransform` struct representing final world transform (x, y, scale, rotation, alpha, depth, width, height).
- `local_transform` — `NodeTransform` struct representing local relative transform.
- `fixed_transform` — `NodeTransform` struct holding starting/unscaled configurations.

## Public Methods

### Hierarchy & Building

#### .node_add(_node1, _node2, ...)
Add child nodes to this parent.
- **Parameters:** One or more child `MiniNode` struct instances.
- **Returns:** Self (for chaining)

#### .node_add_navigation_module()
Initializes a `MiniNodeNavCenter` component to manage children focus.
- **Returns:** Self

#### .node_get_children()
- **Returns:** Array of immediate children.

#### .node_get_all_children()
- **Returns:** Array of all nested descendants (recursively collected).

#### .node_remove(_node)
Remove a child node and unregister it from the manager.
- `_node` — Child node reference to remove.
- **Returns:** Self

#### .node_get_root()
- **Returns:** Topmost ancestor in hierarchy (usually a Canvas).

### Visibility, Activation & Lifecycle

#### .set_visible(_visible)
Toggles rendering visibility of this node.
- `_visible` — Boolean

#### .show() / .hide()
Shorthands to set visibility to `true` or `false`.

## Global NodeManager API
All global functions are prefixed with `mnd_`. Behind the scenes, they fetch variables of `obj_node_manager`.

### Collection management
- `mnd_create_node(_node, _enabled, _root)` — Registers a node using a weak reference. If `_root` is true, registers it to receive automatic updates and drawing cycles from the manager.
- `mnd_remove_node(_node_id)` — De-registers a node.
- `mnd_get_node(_node_id)` — Fetches node struct reference by ID.
- `mnd_enable_node(_node, _grab_focus)` — Enables the node. Optionally moves navigator focus.
- `mnd_disable_node(_node, _delay)` — Disables the node.
- `mnd_node_exists(_node_id)` — Returns boolean.
- `mnd_cleanup_dead_refs()` — Forces garbage collection check.
- `mnd_destroy_all()` — Resets manager.

### Draw Ordering (Roots)
- `mnd_set_root_draw_order(_id, _index)`
- `mnd_bring_root_to_front(_id)`
- `mnd_send_root_to_back(_id)`
- `mnd_get_all_roots()`

### Focus & Navigation Stack
- `mnd_nav_set_focus(_node)` — Shifts active navigation module.
- `mnd_nav_pop_focus()` — Restores focus to the previous menu module in focus history.
- `mnd_nav_get_focused_module()`
- `mnd_nav_has_focus(_node)`
- `mnd_nav_clear_focus()`
- `mnd_get_selected_node()`

### Input Blocking
- `mnd_block_button_navigation(_block)`
- `mnd_block_cursor_navigation(_block)`
- `mnd_block_button_action(_block)`
- `mnd_block_cursor_action(_block)`
- `mnd_block_all_navigation(_block)`
- `mnd_block_all_actions(_block)`
- `mnd_block_all_input(_block)`

### Transitioning
- `mnd_transition(_from_node, _to_node, _delay)` — Disables first node (using delay if requested) and enables target node.

## Input Customization
To overwrite the default keyboard, gamepad, and mouse inputs used by MiniNode, assign new callbacks to the `global.mnd_input` struct:

```gml
global.mnd_input.cursor_x = function() { return my_custom_x; };
global.mnd_input.accept_pressed = function() { return my_custom_press(); };
global.mnd_input.navigate = function() { return [h_axis, v_axis]; };
```
